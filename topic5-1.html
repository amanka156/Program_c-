Поняття алгоритму

Алгоритм — це зрозуміле та точне розпорядження виконавцю зробити послідовність дій, спрямованих на досягнення вказаної мети або розв'язання конкретної задачі.

Термін походить від латинізованого імені перського математика Аль-Хорезмі (Al-Khwarizmi). Алгоритми є основою програмування — будь-яка програма є реалізацією одного або кількох алгоритмів.

🔑 Визначення алгоритму

Алгоритм — це скінченна послідовність чітко визначених інструкцій для розв'язання задачі або досягнення певного результату.

Приклади алгоритмів у повсякденному житті:

  • Рецепт приготування страви
  • Інструкція зі складання меблів
  • Алгоритм знаходження НСД двох чисел (Евклід)
  • Правила розв'язання квадратного рівняння

Величини в алгоритмах

Величина — це елемент даних, який бере участь у виконанні алгоритму. У програмуванні величини реалізуються як змінні та константи.

Властивості величин:

  • Ім'я (ідентифікатор) — унікальне позначення величини
  • Тип — визначає множину допустимих значень та операцій
  • Значення — конкретний стан величини в даний момент

Типи величин:

ЦіліЧисла без дробової частиниint, long, short
ДійсніЧисла з дробовою частиноюfloat, double

Логічніtrue або falsebool

Будь-який алгоритм має володіти такими властивостями:

СкінченністьАлгоритм має завершуватися за скінченну кількість кроків
МасовістьАлгоритм придатний для розв'язання класу задач з різними вхідними даними
РезультативністьАлгоритм має приводити до конкретного результату
КоректністьАлгоритм дає правильний результат для всіх допустимих вхідних даних

Лінійні обчислювальні алгоритми

Лінійний алгоритм — це алгоритм, у якому дії виконуються послідовно одна за одною від початку до кінця. Усі команди виконуються рівно по одному разу.

Основні типи команд у лінійному алгоритмі:

  • Присвоєння — надання змінній значення
  • Введення — отримання даних від користувача
  • Виведення — виведення результату
  • Обчислення — виконання арифметичних операцій

Приклад лінійного алгоритму: Обчислення периметра та площі прямокутника

Лінійний алгоритм
#include <iostream>
using namespace std;

int main() {
    // Введення даних
    double a, b;
    cout << "Введіть довжину прямокутника: ";
    cin >> a;
    cout << "Введіть ширину прямокутника: ";
    cin >> b;
    
    // Обчислення
    double perimeter = 2 * (a + b);
    double area = a * b;
    double diagonal = sqrt(a * a + b * b);
    
    // Виведення результату
    cout << "Периметр: " << perimeter << endl;
    cout << "Площа: " << area << endl;
    cout << "Діагональ: " << diagonal << endl;
    
    return 0;
}

Ще приклади лінійних алгоритмів:

Обмін значень через третю змінну
// Обмін значень двох змінних
int a = 5, b = 10, temp;
temp = a;
a = b;
b = temp;
// Результат: a = 10, b = 5

// Обмін без третьої змінної
a = a + b;  // a = 15
b = a - b;  // b = 5
a = a - b;  // a = 10
Лінійний алгоритм: середнє арифметичне
#include <iostream>
using namespace std;

int main() {
    double num1, num2, num3, average;
    
    cout << "Введіть три числа: ";
    cin >> num1 >> num2 >> num3;
    
    // Лінійний обчислювальний процес
    average = (num1 + num2 + num3) / 3.0;
    
    cout << "Середнє арифметичне: " << average << endl;
    
    return 0;
}

Блок-схеми алгоритмів

Блок-схема — графічне представлення алгоритму з використанням стандартних символів. Вона дозволяє наочно зобразити логіку програми.

СимволФормаПризначення
Початок/КінецьОвалПочаток або кінець алгоритму
Введення/ВиведенняПаралелограмВведення або виведення даних
ПроцесПрямокутникОбчислення, присвоєння
УмоваРомбПеревірка умови (розгалуження)
НапрямокСтрілкаНапрямок виконання
ПідпрограмаПрямокутник з подвійними боковими лініямиВиклик функції

Правила побудови блок-схем:

  • Схема читається зверху вниз та зліва направо
  • Кожна схема має один початок і один кінець
  • З будь-якого блоку виходить тільки одна лінія (крім умови — 2 лінії)
  • До будь-якого блоку входить тільки одна лінія (крім злиття)
  • Лінії можуть зливатися (коло на точці злиття)
✅ Порада

Перед написанням складної програми рекомендується спочатку побудувати блок-схему алгоритму. Це допомагає виявити логічні помилки на ранньому етапі та зробити код більш структурованим.

💻 Практичні завдання

1Блок-схема обміну значень

Побудуйте блок-схему алгоритму обміну значень двох змінних через третю. Потім реалізуйте цей алгоритм на C++.

2Обчислення відстані

Напишіть лінійний алгоритм обчислення відстані між двома точками на площині за координатами (x1, y1) та (x2, y2). d = sqrt((x2-x1)^2 + (y2-y1)^2).

3Перетворення одиниць

Створіть лінійний алгоритм для перетворення: градуси Цельсія → Фаренгейт та Кельвін; кілометри → милі; кілограми → фунти.

4Обчислення зарплати

Напишіть лінійний алгоритм розрахунку зарплати: зарплата = ставка * години + премія(10%) - податок(18%). Виведіть чисту зарплату.

5Геометричні обчислення

Напишіть лінійний алгоритм обчислення площі трикутника за трьома сторонами (формула Герона).

6Час руху

Автомобіль рухається зі швидкістю v1 протягом t1, потім зі швидкістю v2 протягом t2. Напишіть лінійний алгоритм обчислення середньої швидкості.

7Вкладений паралелепіпед

Дано зовнішні розміри паралелепіпеда (a, b, c) та товщину стінки s. Обчисліть об'єм матеріалу стінок. Алгоритм має бути лінійним.

8Електричний ланцюг

За силою струму I та напругою U обчисліть опір R (за законом Ома) та потужність P. Напишіть блок-схему та програму.

9Обмін значень без третьої змінної

Напишіть лінійний алгоритм обміну значень двох цілих змінних без використання третьої змінної (використовуйте + та -).

10Блок-схеми для всіх типів

Побудуйте блок-схеми: а) лінійного алгоритму обчислення суми двох чисел, б) розгалуженого (макс з 2 чисел), в) циклічного (сумма 1 до N). Реалізуйте всі на C++.
