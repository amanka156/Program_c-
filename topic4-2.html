Структура програми на C++

Програма на C++ складається з декількох частин, кожна з яких виконує певну роль:

Структура програми
// 1. Коментарі — пояснення для програміста
/* Це приклад структури програми на C++ */

// 2. Директиви препроцесора
#include <iostream>    // підключення бібліотек
#include <cmath>
using namespace std;      // використання простору імен

// 3. Оголошення глобальних констант
const double PI = 3.14159;
const int MAX_SIZE = 1000;

// 4. Оголошення (прототипи) функцій
void helperFunction();
int calculate(int a, int b);

// 5. Головна функція — точка входу в програму
int main() {
    // Локальні змінні
    int x = 10;
    double y = 3.14;
    
    // Виклики функцій
    helperFunction();
    int result = calculate(x, 5);
    
    return 0;  // код завершення
}

// 6. Визначення функцій
void helperFunction() {
    cout << "Допоміжна функція" << endl;
}

int calculate(int a, int b) {
    return a + b;
}

Основні складові частини:

ЧастинаПризначення
КоментаріДокументація та пояснення коду
Директиви препроцесораПідключення бібліотек, визначення макросів
Глобальні оголошенняКонстанти, змінні, прототипи функцій
Функція main()Головна функція — точка входу
Функції користувачаПідпрограми для розв'язування підзадач

Глобальні та локальні змінні

Глобальна змінна — оголошена поза всіма функціями, доступна з будь-якої частини програми (включаючи інші файли через extern).

Локальна змінна — оголошена всередині блоку коду (функції, циклу, умови), доступна тільки в цьому блоці.

Глобальні та локальні змінні
#include <iostream>
using namespace std;

// Глобальні змінні
int globalVar = 100;
const double PI = 3.14159;

void testFunction() {
    // Доступ до глобальної змінної
    cout << "globalVar в функції: " << globalVar << endl;
    
    // Локальна змінна
    int localVar = 50;
    cout << "localVar: " << localVar << endl;
}

int main() {
    cout << "globalVar в main: " << globalVar << endl;
    
    // Локальна змінна з тим самим ім'ям
    int globalVar = 200;   // приховує глобальну!
    cout << "Локальна globalVar: " << globalVar << endl;
    cout << "Глобальна globalVar: " << ::globalVar << endl; // :: — доступ до глобальної
    
    testFunction();
    
    return 0;
}
⚠️ Попередження

Використання глобальних змінних не рекомендується, оскільки вони:

  • Ускладнюють відлагодження та тестування
  • Створюють неявні залежності між функціями
  • Погіршують повторне використання коду
  • Можуть призвести до конфліктів імен

Краще передавати дані через параметри функцій та повертати результати.

Області видимості (Scope)

Область видимості — це частина програми, де змінна доступна для використання.

Типи областей видимості в C++:

ТипОписПриклад
Локальна (блочна)Змінна видима в блоці {}Змінна всередині функції
Глобальна (файлова)Змінна видима у всьому файліЗмінна поза функціями
Простір іменВидимість у межах namespacestd::cout
Параметри функціїВидимість у тілі функціїПараметри в заголовку
Області видимості
#include <iostream>
using namespace std;

int x = 10;  // глобальна змінна

int main() {
    int x = 20;   // локальна змінна — приховує глобальну
    
    {
        int x = 30;   // вкладена локальна змінна
        cout << "Внутрішній блок: x = " << x << endl;       // 30
        cout << "Глобальна: x = " << ::x << endl;           // 10
    }
    
    cout << "Функція main: x = " << x << endl;              // 20
    
    return 0;
}

Класи зберігання (Storage Classes)

Класи зберігання визначають час життя та область видимості змінних.

КласКлючове словоОпис
АвтоматичнийautoЛокальна змінна, створюється при вході в блок, знищується при виході (за замовчуванням)
СтатичнийstaticЗберігає значення між викликами функції; для глобальних — обмежує видимість файлом
ЗовнішнійexternОголошення змінної, визначеної в іншому файлі
РеєстровийregisterПропозиція компілятору зберігати змінну в регістрі (застаріле)

Приклад static змінної:

Static змінна
#include <iostream>
using namespace std;

void counter() {
    static int count = 0;   // ініціалізується тільки один раз!
    count++;
    cout << "Виклик номер: " << count << endl;
}

int main() {
    counter();  // Виклик номер: 1
    counter();  // Виклик номер: 2
    counter();  // Виклик номер: 3
    return 0;
}

Час життя змінних

Тип змінноїЧас життяІніціалізація
Локальна (auto)Від входу до виходу з блокуНе ініціалізується автоматично (сміття)
Статична локальнаВід першого виклику до завершення програмиІніціалізується один раз
ГлобальнаВід запуску до завершення програмиАвтоматично нулем
Статична глобальнаВід запуску до завершення програмиАвтоматично нулем; видимість тільки у файлі
ДинамічнаКерується програмістом (new/delete)Не ініціалізується

Багатофайлові програми

Великі програми розбиваються на кілька файлів для зручності розробки та компіляції:

  • Заголовкові файли (.h, .hpp) — прототипи функцій, оголошення класів, константи
  • Файли реалізації (.cpp) — визначення функцій, глобальні змінні
  • Головний файл (main.cpp) — функція main(), яка використовує інші модулі
mathutils.h — заголовковий файл
#ifndef MATHUTILS_H
#define MATHUTILS_H

// Прототипи функцій
int add(int a, int b);
int multiply(int a, int b);
double power(double base, int exp);

// Константи
const double PI = 3.14159265359;

#endif
mathutils.cpp — файл реалізації
#include "mathutils.h"

int add(int a, int b) {
    return a + b;
}

int multiply(int a, int b) {
    return a * b;
}

double power(double base, int exp) {
    double result = 1.0;
    for (int i = 0; i < exp; i++)
        result *= base;
    return result;
}
main.cpp — головний файл
#include <iostream>
#include "mathutils.h"
using namespace std;

int main() {
    cout << "2 + 3 = " << add(2, 3) << endl;
    cout << "2 * 3 = " << multiply(2, 3) << endl;
    cout << "2^5 = " << power(2, 5) << endl;
    cout << "PI = " << PI << endl;
    return 0;
}

💻 Практичні завдання

1Лічильник викликів

Створіть функцію, яка рахує кількість своїх викликів за допомогою static змінної. Виведіть лічильник після 10 викликів.

2Багатофайловий проєкт

Розбийте програму на 3 файли: utils.h (прототипи), utils.cpp (реалізація), main.cpp (використання). Функції: min, max, abs.

3Області видимості

Напишіть програму з 3 рівнями вкладеності блоків {}, на кожному рівні оголосіть змінну з однаковим ім'ям. Виведіть значення кожної змінної та доступ до глобальної через ::.

4Static масив у функції

Реалізуйте функцію, яка зберігає static масив із 5 чисел. При кожному виклику виводить усі значення масиву та додає нове число (зсув масиву).

5Генератор унікальних ID

Створіть функцію getNextID(), яка використовує static лічильник для генерації унікальних ідентифікаторів (1, 2, 3...).

6Протект від повторного включення

Створіть заголовковий файл з використанням #ifndef/#define/#endif та з guard #pragma once. Покажіть на прикладі, чому це важливо.

7Extern змінна

Створіть два .cpp файли. У одному оголосіть глобальну змінну, в іншому — отримайте до неї доступ через extern. Покажіть як це працює.

8Аналіз пам'яті

Напишіть програму, яка демонструє розташування різних типів змінних у пам'яті: глобальні, static, локальні, динамічні. Виведіть їх адреси.

9Модульна бібліотека

Створіть модульну структуру: math.h/cpp (математика), string_utils.h/cpp (робота з рядками), main.cpp. Кожен модуль у своєму просторі імен.

10Проблема глобальних змінних

Напишіть програму, яка демонструє проблему використання глобальних змінних: створіть ситуацію, коли різні функції неочікувано змінюють глобальний стан. Запропонуйте рішення без глобальних змінних.
